# Dashboard Installation & Running Guide

## Phase 3: Web Visualization for Okta Security Monitoring Dashboard

This guide covers how to set up and run the interactive web dashboard for the Okta Security Monitoring project.

### What's Included

The dashboard adds a professional web interface to visualize the analysis results generated by the Phase 2 analysis engine.

**New Files:**
- `dashboard.py` - Flask web server and API endpoints
- `templates/dashboard.html` - Responsive HTML dashboard
- `static/css/style.css` - Professional styling with dark mode support
- `static/js/dashboard.js` - Chart.js visualization and interactivity

### Prerequisites

- Python 3.8 or higher
- Virtual environment activated (from Phase 1/2 setup)
- Analysis results generated by running `python src/main.py`

### Step 1: Install Flask Dependency

Flask is already included in requirements.txt. If you haven't installed it yet:

```bash
pip install -r requirements.txt
```

Or install Flask directly:

```bash
pip install Flask==3.0.0
```

### Step 2: Generate Analysis Data (if needed)

The dashboard reads from JSON files generated by the analysis engine. First, generate fresh data:

```bash
# From the project root directory
python src/main.py
```

This creates a file like `analysis_results_20260111_143022.json` in the project root.

### Step 3: Run the Dashboard Server

```bash
# From the project root directory
python dashboard.py
```

You should see output like:

```
 * Running on http://127.0.0.1:5000
 * Press CTRL+C to stop the server
```

### Step 4: Access the Dashboard

Open your web browser and navigate to:

```
http://localhost:5000
```

### API Endpoints

The dashboard server provides the following API endpoints for data integration:

```bash
# Get complete dashboard data
curl http://localhost:5000/api/analysis

# Get summary statistics only
curl http://localhost:5000/api/summary

# Get threat data
curl http://localhost:5000/api/threats

# Get MFA analysis
curl http://localhost:5000/api/mfa

# Get geographic patterns
curl http://localhost:5000/api/geography

# Get server status
curl http://localhost:5000/api/status
```

### Deployment Options

#### Development (Local Testing)

```bash
python dashboard.py
```

Access at `http://localhost:5000`

#### Production with Gunicorn

```bash
pip install gunicorn
gunicorn -w 4 -b 0.0.0.0:5000 dashboard:app
```

#### Production with Docker

Create `Dockerfile`:

```dockerfile
FROM python:3.12-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install -r requirements.txt
COPY . .
CMD ["python", "dashboard.py"]
```

Build and run:

```bash
docker build -t okta-dashboard .
docker run -p 5000:5000 okta-dashboard
```

### Troubleshooting

#### Problem: "No analysis results found"

**Solution**: Run the analysis first before starting the dashboard:

```bash
python src/main.py
# Then
python dashboard.py
```

#### Problem: Port 5000 already in use

**Solution**: Use a different port:

```bash
# Modify dashboard.py last line:
app.run(debug=True, host='0.0.0.0', port=8080)
```

Or kill the existing process:

```bash
# Linux/Mac
lsof -ti:5000 | xargs kill -9

# Windows
netstat -ano | findstr :5000
taskkill /PID <PID> /F
```

#### Problem: Templates or static files not found

**Solution**: Make sure you're running the script from the project root directory:

```bash
cd /path/to/okta-security-monitoring-dashboard
python dashboard.py
```

#### Problem: Charts not displaying

**Solution**: Check browser console for errors (F12 â†’ Console). Ensure JavaScript is enabled and CDN resources are loading.
